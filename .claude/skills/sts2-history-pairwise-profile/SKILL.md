# StS2 Pairwise Profile

Use this skill for stage 5: same-screen card choice evidence.

## Inputs

- V1: `10_card_pairwise_choices.csv`, `11_card_pairwise_summary.csv`, `12_card_pairwise_all_non_shop_choices.csv`, `13_card_pairwise_all_non_shop_summary.csv`
- V2: `14_pairwise_frequent.csv`, `15_pairwise_dominant.csv`, `16_pairwise_controversial.csv`, `17_pairwise_act_reversal.csv`, `18_pairwise_card_profile.csv`

## Task Flow

1. Apply minimum co-occurrence gates before reading a pair as evidence.
2. Separate standard reward pairwise from all non-shop pairwise.
3. Identify A-over-B candidates, A-loses-to-B candidates, frequent matchups, and controversial matchups.
4. Check act-level reversal before writing a universal matchup rule.
5. Produce candidate rules with exact pair, sample count, win rate, act split, counterexample rows, and strength.

## Code Boundary

Pairwise tables and reversal candidates should be generated by Python. If the current tables cannot express the matchup claim, add a reusable V2 table rather than inferring from prose.

## Reasoning Boundary

Pairwise choice is context-dependent. Review whether the pair reflects real preference, deck need, rarity, reward source, or act pressure before accepting a rule.
